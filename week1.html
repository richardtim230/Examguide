Day 1 (Thursday): HTML Basics & Your First Page

What's Happening Today?

We're kicking off with the absolute basics. You'll learn what HTML is, what a web browser does, and how to create the fundamental structure of every web page. Think of it like learning how to lay the foundation and frame for a house.

Concepts – Pay Attention to This:

What is Web Development?

  • Client-side: This is what you see in your browser (HTML, CSS, JavaScript). It runs on your computer.
  • Server-side: This is what happens behind the scenes (databases, logic, sending data to your browser). It runs on a server. We're focusing on client-side for now.

What is HTML?

  • HTML stands for HyperText Markup Language.
  • It's not a programming language. It's a "markup" language. It tells the browser what content is (e.g., "this is a heading," "this is a paragraph," "this is an image").
  • It forms the structure and content of your webpage.

The Browser's Job:

  • Your web browser (Chrome, Firefox, Safari, Edge) reads HTML files.
  • It then interprets that HTML and displays the page visually for you.

Basic HTML Document Structure:

Every HTML page starts the same way. Memorize this!

<!DOCTYPE html>  <!-- Tells the browser this is an HTML5 document -->
<html lang="en"> <!-- The root element of the page. 'lang="en"' sets the language -->
<head>           <!-- Contains metadata (info about the page), not visible on the page itself -->
    <meta charset="UTF-8"> <!-- Specifies character encoding (important for all characters to display correctly) -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Good for responsive design (later) -->
    <title>Your Page Title Here</title> <!-- The text that appears in the browser tab -->
</head>
<body>           <!-- Contains all the visible content of your web page -->

    <!-- All your visible content goes here -->

</body>
</html>

Key Terms:

  • Tags: HTML uses "tags" enclosed in angle brackets (e.g., <body>). Most tags come in pairs: an opening tag (<p>) and a closing tag (</p>). Content goes between them. Some tags are "self-closing" (<br>, <hr>).
  • Elements: An HTML element consists of the opening tag, the closing tag, and all the content in between.
  • Attributes: Tags can have attributes which provide extra information about the element. They look like attribute="value" (e.g., lang="en", charset="UTF-8").

Common Block-Level Elements (They take up their own line):

  • Headings (<h1> to <h6>): Used for titles and subtitles. <h1> is the most important (main title), <h6> is the least. They are semantic, meaning they indicate the importance and structure of your content, not just size.
  • Example: <h1>Main Title</h1>
  • Paragraphs (<p>): Used for blocks of regular text.
  • Example: <p>This is a paragraph of text.</p>
  • Line Break (<br>): Forces a new line. It's self-closing. Use sparingly. Don't use it for spacing out paragraphs (CSS is for that).
  • Example: Line one.<br>Line two.
  • Horizontal Rule (<hr>): Draws a horizontal line. It's self-closing. Used to indicate a thematic break in content.
  • Example: <hr>

Your Hands-On Activity: Create Your First Bio Page

Set Up Your Workspace:

  • Install VS Code: If you haven't, download and install Visual Studio Code (VS Code). It's a free, powerful code editor.
  • Install "Live Server" Extension: In VS Code, go to the Extensions view (Ctrl+Shift+X or Cmd+Shift+X), search for "Live Server" by Ritwick Dey, and install it. This will let you see your changes instantly in the browser.

Create Your File:

  • Open VS Code.
  • Go to File > New File.
  • Save it immediately as index.html in a new folder (e.g., "my-first-website"). Make sure the .html extension is there!

Add the Basic Structure:

  • In your empty index.html file, type ! (exclamation mark) and press Tab. VS Code will generate the basic HTML structure for you.
  • Change the <title> tag's content to "My First Bio Page".

Add Your Content (inside the <body> tags):

  • Your Name: Add your full name as an <h1> heading.
    • <h1>Your Full Name</h1>
  • About Me Paragraph: Write a short paragraph about yourself using <p>.
    • <p>Hi there! My name is Your Name, and I'm excited to learn web development. I love [mention a hobby or interest].</p>
  • A New Section: Add an <h2> heading called "My Interests".
    • <h2>My Interests</h2>
  • Another Paragraph: Write another paragraph about your interests.
    • <p>I am passionate about [interest 1]. I also enjoy [interest 2].</p>
  • Use Line Breaks and Horizontal Rule:
    • <p>This is some text.<br>And this is on a new line.</p>
<hr>
<p>This is more text after a horizontal line.</p>

View Your Page:

  • In VS Code, right-click anywhere in your index.html file and select "Open with Live Server".
  • Your default browser will open and display your page!
  • Any changes you save in VS Code will now automatically update in your browser.

Day 2 (Friday): Text Formatting & Lists

What's Happening Today?

You'll learn how to make your text look different (bold, italic, etc.), organize information using various types of lists, and add interactive links and images to your page.

Concepts – Pay Attention to This:

Text Formatting Elements:

  • <strong> and <b>:
    • <strong>: Semantic, indicates text with strong importance (e.g., warnings).
    • <b>: Just bold visually, no extra importance.
    • Best practice: Use <strong> when the text is important, otherwise use CSS for bolding.
  • <em> and <i>:
    • <em>: Semantic, indicates emphasized text (e.g., spoken emphasis).
    • <i>: Just italic visually, for things like technical terms, foreign words.
    • Best practice: Use <em> for emphasis, otherwise use CSS for italicizing.
  • <u>: Underlined text (use sparingly, as it can be confused with links).
  • <sup>: Superscript text (e.g., 2<sup>nd</sup>).
  • <sub>: Subscript text (e.g., H<sub>2</sub>O).

List Elements:

  • Unordered List (<ul>):
    • Used for bulleted lists where the order doesn't matter.
    • Each item in the list is a <li> (list item).
    • Example:
      • <ul>
    <li>Milk</li>
    <li>Eggs</li>
    <li>Bread</li>
</ul>
  • Ordered List (<ol>):
    • Used for numbered or lettered lists where the order does matter (e.g., steps in a recipe).
    • Each item is also a <li>.
    • Example:
      • <ol>
    <li>Preheat oven.</li>
    <li>Mix ingredients.</li>
    <li>Bake for 30 minutes.</li>
</ol>
  • Definition List (<dl>):
    • Used for terms and their definitions/descriptions.
    • <dt> (definition term) and <dd> (definition description).
    • Example:
      • <dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language, the language for structuring web content.</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets, the language for styling web content.</dd>
</dl>

Link Element (<a> - Anchor Tag):

  • Used to create hyperlinks to other pages or resources.
  • The href attribute specifies the destination URL.
  • Absolute URLs: For external websites (e.g., https://www.google.com).
  • Relative URLs: For pages within your own website (e.g., about.html).
  • target="_blank" attribute: Opens the linked document in a new browser tab.
  • Example: <a href="https://www.google.com" target="_blank">Go to Google</a>

Image Element (<img>):

  • Used to embed images. It's a self-closing tag.
  • src attribute: Specifies the path to the image file. Can be a URL or a relative path (e.g., images/my-photo.jpg).
  • alt attribute: Crucial! Provides alternative text for the image.
    • Displayed if the image can't load.
    • Read by screen readers for visually impaired users.
    • Important for SEO (Search Engine Optimization).
  • width and height attributes: Can set the size of the image in pixels. (We'll use CSS for better control later, but these are good for initial sizing).
  • Example: <img src="my-photo.jpg" alt="A photo of me smiling" width="300" height="200">

Day 3 (Saturday): Semantic HTML & Tables

What's Happening Today?

Today, you'll learn about using HTML in a smarter way (semantic HTML) to give meaning to your page sections, not just visual layout. You'll also learn how to create proper data tables.

Concepts – Pay Attention to This:

Semantic HTML5 Elements – Giving Meaning to Your Content:

  • Before HTML5, we used <div> elements for almost everything. But <div> doesn't tell anyone what the content is.
  • Semantic HTML5 tags describe the purpose of the content they enclose. This is crucial for:
    • Accessibility: Screen readers can better navigate pages.
    • SEO (Search Engine Optimization): Search engines understand your content better.
    • Readability: Your code is easier for other developers (and you!) to understand.

Key Semantic Elements:

  • <header>: Introduces content, typically at the top of a document or a section. Often contains a logo, site title, and navigation.
  • <nav>: Contains navigation links (menus).
  • <main>: The dominant content of the <body>. There should only be one <main> element per page.
  • <article>: Represents self-contained, independent content that could be distributed independently (e.g., a blog post, a news story, a comment).
  • <section>: A thematic grouping of content, typically with a heading. Think of it as a distinct section within an article or page.
  • <aside>: Contains content that is tangentially related to the content around it, often presented as a sidebar.
  • <footer>: Contains authorship information, copyright data, related documents, etc. Typically at the bottom of a document or a section.

Non-Semantic Elements (Still useful!):

  • <div>: A generic block-level container. Use when no other semantic element is appropriate (e.g., for grouping elements for styling purposes).
  • <span>: A generic inline container. Use for styling small parts of text within a line.

Basic Table Elements:

  • Tables are for tabular data (rows and columns of data), NOT for page layout (we'll use CSS for layout later).
  • <table>: The main container for the entire table.
  • <thead>: Groups the header content in a table (e.g., column titles).
  • <tbody>: Groups the main body content of a table.
  • <tfoot>: Groups the footer content in a table (e.g., sums or totals).
  • <tr>: Table Row. Each <tr> defines a new row in the table.
  • <th>: Table Header cell. Content inside is typically bold and centered by default, indicates a header for a column or row.
  • <td>: Table Data cell. Contains the actual data for the table.

Attributes for Merging Cells:

  • colspan: Makes a cell span across multiple columns.
  • rowspan: Makes a cell span across multiple rows.

Example Table Structure:

<table>
    <thead>
        <tr>
            <th>Day</th>
            <th>Activity</th>
            <th>Time</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Monday</td>
            <td>Coding Class</td>
            <td>9:00 AM</td>
        </tr>
        <tr>
            <td>Tuesday</td>
            <td>Gym</td>
            <td>6:00 PM</td>
        </tr>
    </tbody>
</table>

Your Hands-On Activity: Semantic Bio Page & Your First Table

Re-structure Your Bio Page Semantically:

  • Open your index.html file with Live Server.
  • Look at your page and decide which semantic HTML5 tags fit best for each section.
  • Wrap your entire page content (inside <body>) with <main> tags.
  • Identify your main name/title area: Wrap it in a <header> tag.
  • If you had navigation links (you will soon!), they would go in a <nav> tag.
  • Group your "About Me" and "My Interests" sections with <section> tags. Give each section an <h2> heading.
  • Add a <footer> tag at the bottom for copyright info or a simple note.
  • Example (Conceptual):
    • <body>
    <header>
        <h1>Your Name</h1>
        <p>Welcome to my page!</p>
    </header>

    <main>
        <section>
            <h2>About Me</h2>
            <p>...</p>
        </section>

        <section>
            <h2>My Interests</h2>
            <ul>...</ul>
        </section>
    </main>

    <footer>
        <p>© 2024 Your Name. All rights reserved.</p>
    </footer>
</body>
  • Note: You won't see any visual changes yet! Semantic HTML is about meaning, not appearance.

Create a Simple Table:

  • In your index.html (or create a new table.html if you prefer to keep index.html clean for your project), create a simple table.
  • Idea 1: Daily Schedule (Simple):
    • Columns: Day, Activity, Time.
    • Include <thead> and <tbody>.
  • Idea 2: Product Comparison (Slightly More Complex):
    • Columns: Feature, Product A, Product B.
    • Include <thead> and <tbody>.
  • Experiment with colspan and rowspan (Optional, but try it!):
    • Can you make a cell span across two columns (e.g., for a "Notes" section that goes across the bottom)?
    • Can you make a cell span down two rows (e.g., if a person has two activities at the same time)?

Your Homework:

  • Refine Your Multi-Section Page Idea: Remember that simple portfolio or product page idea from Day 1 homework?
  • Outline HTML Structure: On paper or in a simple text file, sketch out the main HTML structure using the semantic HTML5 elements you just learned.
  • Think: What will be your <header>? What's the <nav>? What are the main <section>s or <article>s? What belongs in the <footer>?
  • Don't write any content yet, just the nested tags. This is about planning the skeleton of a bigger project.

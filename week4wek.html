Day 1 (Thursday): Transitions, Transforms & Basic Animations

What's Happening Today?

Today, you'll learn how to add smooth transitions, visual transformations, and basic animations to make your web pages more dynamic and engaging.

Concepts – Pay Attention to This:

CSS Transitions:

  • Transitions: Smoothly change property values over time when triggered (e.g., on hover).
  • transition-property: Specifies which CSS properties to transition (e.g., background-color, all).
  • transition-duration: Sets the duration (e.g., 0.3s, 300ms).
  • transition-timing-function: Controls the speed curve (e.g., ease, linear, ease-in-out).
  • transition-delay: Delays the start of the transition (e.g., 0.1s).
  • Shorthand transition: Combines all properties (e.g., transition: background-color 0.3s ease;).
  • Example:
    • button {
    background-color: #3498db;
    transition: background-color 0.3s ease;
}
button:hover {
    background-color: #1f6391;
}

CSS Transforms:

  • Transforms: Modify an element's appearance or position (e.g., rotate, scale, move).
  • translate(x, y): Moves an element (e.g., translate(10px, 20px)).
  • rotate(deg): Rotates an element (e.g., rotate(45deg)).
  • scale(x, y): Resizes an element (e.g., scale(1.2) for 20% larger).
  • skew(x, y): Skews an element along the x/y axis (e.g., skew(10deg, 0)).
  • Example:
    • .image {
    transition: transform 0.3s ease;
}
.image:hover {
    transform: scale(1.1) rotate(5deg);
}

Basic CSS Animations:

  • @keyframes: Defines the animation steps.
  • Animation Properties:
    • animation-name: Links to a @keyframes rule.
    • animation-duration: Sets the duration.
    • animation-iteration-count: Number of times to repeat (e.g., infinite, 3).
    • animation-timing-function: Like transitions (e.g., ease).
  • Shorthand animation: Combines properties.
  • Example:
    • @keyframes pulse {
    0% { transform: scale(1); }
    50% { transform: scale(1.1); }
    100% { transform: scale(1); }
}
.element {
    animation: pulse 2s infinite ease-in-out;
}
  • Note: Use animations sparingly to avoid overwhelming users. Test with Chrome DevTools to fine-tune timing.

Your Hands-On Activity: Add Interactivity

Add Transition Effects:

  • Open your index.html or recipe.html and its CSS file.
  • Add a button or link with a hover transition:
    • <button class="cta-button">Learn More</button>
    .cta-button {
    background-color: #3498db;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    transition: background-color 0.3s ease, transform 0.2s ease;
}
.cta-button:hover {
    background-color: #1f6391;
    transform: translateY(-2px);
}

Apply a Transform:

  • Add a transform to an image on hover:
    • <img src="my-image.jpg" alt="Description" class="hover-image">
    .hover-image {
    transition: transform 0.3s ease;
}
.hover-image:hover {
    transform: scale(1.05) rotate(2deg);
}

Create a Pulsating Effect:

  • Add a pulsating animation to an element (e.g., a badge or icon):
    • <div class="badge">New!</div>
    @keyframes pulse {
    0% { transform: scale(1); }
    50% { transform: scale(1.1); }
    100% { transform: scale(1); }
}
.badge {
    background-color: #e74c3c;
    color: white;
    padding: 5px 10px;
    border-radius: 5px;
    animation: pulse 1.5s infinite ease-in-out;
}

Your Homework:

  • Find Transition/Transform Examples: Browse websites and identify at least three examples of transitions or transforms (e.g., buttons that change color, images that zoom).
  • Take notes on what properties are used (use Chrome DevTools to inspect).
  • Try replicating one effect in your recipe.html or another page.

Day 2 (Friday): Forms & Pseudo-Classes/Elements

What's Happening Today?

You'll learn how to style HTML forms to make them user-friendly and visually appealing, and use pseudo-classes and pseudo-elements to add dynamic and decorative effects.

Concepts – Pay Attention to This:

Styling Forms:

  • Form Elements: Common elements include <input>, <textarea>, <select>, and <button>.
  • <input> Types: text, email, password, submit, etc.
  • Styling Tips:
    • Use padding and borders for clarity.
    • Ensure sufficient contrast for readability.
    • Style :focus states for accessibility.
  • Example:
    • input[type="text"], textarea {
    width: 100%;
    padding: 10px;
    border: 1px solid #ccc;
    border-radius: 5px;
    font-size: 16px;
}
input[type="submit"] {
    background-color: #3498db;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    cursor: pointer;
}

Pseudo-Classes:

  • Pseudo-Classes: Style elements based on their state or position.
  • Common ones:
    • :hover: When the mouse hovers over an element.
    • :active: When an element is being clicked.
    • :focus: When an element is selected (e.g., input field).
    • :nth-child(n): Targets elements based on their position (e.g., :nth-child(2) for the second child).
    • :first-child, :last-child: Target the first or last child.
  • Example:
    • input:focus {
    border-color: #2980b9;
    outline: none;
}
li:nth-child(odd) {
    background-color: #f0f3f5;
}

Pseudo-Elements:

  • Pseudo-Elements: Style specific parts of an element.
  • Common ones:
    • ::before: Inserts content before an element's content.
    • ::after: Inserts content after an element's content.
  • Requires content property (even if empty, e.g., content: '';).
  • Example:
    • h2::before {
    content: '★ ';
    color: #e74c3c;
}
.button::after {
    content: ' →';
    margin-left: 5px;
}

Your Hands-On Activity: Create and Style a Contact Form

Create a Form:

  • Create a new HTML file (e.g., contact.html) and CSS file.
  • Add a contact form:
    • <section class="contact-form">
    <h2>Contact Us</h2>
    <form>
        <label for="name">Name:</label>
        <input type="text" id="name" name="name" required>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
        <label for="message">Message:</label>
        <textarea id="message" name="message" rows="5"></textarea>
        <button type="submit">Send</button>
    </form>
</section>

Style the Form:

  • In your CSS file:
    • .contact-form {
    max-width: 500px;
    margin: 0 auto;
    padding: 20px;
}
label {
    display: block;
    margin-bottom: 5px;
    font-weight: bold;
}
input, textarea {
    width: 100%;
    padding: 10px;
    margin-bottom: 15px;
    border: 1px solid #ccc;
    border-radius: 5px;
    box-sizing: border-box;
}
input:focus, textarea:focus {
    border-color: #2980b9;
    outline: none;
}
button {
    background-color: #3498db;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    cursor: pointer;
    transition: background-color 0.3s ease;
}
button:hover {
    background-color: #1f6391;
}

Add Pseudo-Elements:

  • Add a decorative element to your form heading:
    • .contact-form h2::before {
    content: '✉ ';
    color: #e74c3c;
}
  • Add an arrow after the submit button:
    • button::after {
    content: ' →';
    margin-left: 5px;
}

Your Homework:

  • Design a Login/Registration Form: Create a new HTML/CSS file pair.
  • Include inputs for username, password, and a submit button.
  • Style it with consistent padding, borders, and focus states.
  • Use pseudo-classes like :hover and :focus for interactivity.
  • Add a pseudo-element (e.g., ::before for a decorative icon).

Day 3 (Saturday): Final Project & Review

What's Happening Today?

Today, you'll review HTML and CSS best practices, debug with DevTools, and work on a multi-page website project (e.g., portfolio, product landing, or blog) that showcases everything you've learned.

Concepts – Pay Attention to This:

HTML & CSS Review:

  • Semantic HTML: Use tags like <header>, <main>, <section>, and <footer> for meaningful structure.
  • External Stylesheets: Keep CSS in separate .css files for maintainability.
  • Naming Conventions: Use clear, consistent class/ID names (e.g., .navbar, .hero-section).
  • Basic Accessibility:
    • Use alt attributes for images.
    • Ensure sufficient color contrast.
    • Use semantic tags for screen readers.
    • Add aria-label where needed (e.g., for buttons with no text).

Debugging with DevTools:

  • Chrome DevTools: Right-click an element and select "Inspect" to open.
  • Key Features:
    • Elements Tab: View and edit HTML/CSS in real-time.
    • Styles Pane: See applied styles and toggle properties.
    • Computed Tab: View final calculated styles (e.g., box model).
    • Console: Check for errors or warnings.
  • Example: If a margin isn't working, check if a parent element's overflow or display is affecting it.

Your Hands-On Activity: Build Your Multi-Page Website

Plan Your Project:

  • Revisit your multi-section page idea from Week 1 (e.g., portfolio, product landing, blog).
  • Create a folder structure:
    • my-website/
├── index.html
├── about.html
├── contact.html
├── css/
│   └── styles.css
└── images/
  • Pages to include:
    • index.html: Home page with a hero section and overview.
    • about.html: About page with your bio or product details.
    • contact.html: Contact form from Day 2.

Build the Structure:

  • Use semantic HTML for each page:
    • <header>
    <h1>My Website</h1>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>
            <li><a href="about.html">About</a></li>
            <li><a href="contact.html">Contact</a></li>
        </ul>
    </nav>
</header>
<main>
    <section class="hero">
        <h2>Welcome</h2>
        <p>...</p>
    </section>
    <section class="content">
        <!-- Page-specific content -->
    </section>
</main>
<footer>
    <p>© 2025 Your Name</p>
</footer>

Style with All Learned Techniques:

  • In styles.css, apply:
    • Layout: Use Flexbox or Grid for page structure (e.g., navigation, content areas).
    • Styling: Use colors, fonts, and box model properties from Week 2.
    • Interactivity: Add transitions/transforms from Day 1.
    • Forms: Style navigation links or form elements with pseudo-classes/elements.
  • Example:
    • * {
    box-sizing: border-box;
}
body {
    margin: 0;
    font-family: 'Arial', sans-serif;
}
header {
    display: flex;
    justify-content: space-between;
    align-items: center;
    background-color: #2c3e50;
    color: white;
    padding: 20px;
}
nav ul {
    display: flex;
    list-style: none;
}
nav a {
    color: white;
    text-decoration: none;
    padding: 10px;
    transition: color 0.3s ease;
}
nav a:hover {
    color: #3498db;
}
.hero {
    background: url('images/hero.jpg') no-repeat center/cover;
    padding: 50px;
    text-align: center;
}
.hero h2 {
    font-size: 36px;
    animation: fadeIn 1s ease-in;
}
@keyframes fadeIn {
    0% { opacity: 0; }
    100% { opacity: 1; }
}

Debug and Test:

  • Use Chrome DevTools to:
    • Check for overlapping elements (use z-index if needed).
    • Ensure responsiveness by testing in the "Device Toolbar" mode.
    • Verify accessibility (e.g., sufficient contrast, alt attributes).
  • Test navigation links to ensure they work across pages.

Your Homework:

  • Polish Your Website: Add at least one new feature (e.g., a hover effect, an animation, or a styled form).
  • Review Accessibility: Check your site with a screen reader (e.g., NVDA or VoiceOver) or use an online accessibility checker.
  • Q&A Prep: Write down any questions about HTML/CSS or issues you encountered for a Q&A session.
